📚 Conceitos Fundamentais

O que é HTML Semântico?

HTML semântico refere-se ao uso de elementos HTML que têm significado e propósito específico, não apenas aparência visual. Estes elementos comunicam a estrutura e o significado do conteúdo tanto para desenvolvedores quanto para tecnologias assistivas.

Atualizado em 15 de março de 2024

✅ Benefícios do HTML Semântico:

  • SEO melhorado: Motores de busca compreendem melhor o conteúdo
  • Acessibilidade: Leitores de ecrã navegam mais facilmente
  • Manutenibilidade: Código mais claro e organizad
  • Performance: Carregamento e processamento mais eficiente
  • Compatibilidade: Melhor suporte em diferentes dispositivos

⚖️ Semântico vs Não-semântico

❌ Não-semântico (antigo)

<div id="header"> <div class="title">Título</div> </div> <div id="nav"> <div class="menu">Menu</div> </div> <div id="content"> <div class="article"> Conteúdo do artigo </div> </div> <div id="sidebar"> Barra lateral </div> <div id="footer"> Rodapé </div>

✅ Semântico (moderno)

<header> <h1>Título</h1> </header> <nav> <ul> <li>Menu item</li> </ul> </nav> <main> <article> <h2>Título do artigo</h2> <p>Conteúdo do artigo</p> </article> </main> <aside> Barra lateral </aside> <footer> Rodapé </footer>

🧱 Elementos Semânticos HTML5

Elementos de Estrutura

<header>

Propósito: Cabeçalho de página ou seção

Conteúdo típico: Logo, título, navegação principal

<header> <h1>Nome do Site</h1> <nav>...</nav> </header>

<nav>

Propósito: Seção de navegação

Conteúdo típico: Menus, links de navegação

<nav role="navigation"> <ul> <li><a href="#home">Home</a></li> <li><a href="#about">Sobre</a></li> </ul> </nav>

<main>

Propósito: Conteúdo principal da página

Nota: Apenas um <main> por página

<main> <h1>Título Principal</h1> <article>...</article> </main>

<article>

Propósito: Conteúdo independente e reutilizável

Exemplos: Post de blog, notícia, comentário

<article> <header> <h2>Título do Artigo</h2> <time datetime="2024-03-15">15 Mar 2024</time> </header> <p>Conteúdo do artigo...</p> </article>

<section>

Propósito: Seção temática do conteúdo

Nota: Deve ter um cabeçalho (h1-h6)

<section> <h2>Produtos</h2> <p>Lista de produtos...</p> </section>

<aside>

Propósito: Conteúdo relacionado mas secundário

Exemplos: Sidebar, widgets, anúncios

<aside> <h3>Artigos Relacionados</h3> <ul> <li><a href="#">Artigo 1</a></li> </ul> </aside>

<footer>

Propósito: Rodapé de página ou seção

Conteúdo típico: Copyright, links, contactos

<footer> <p>© 2024 Empresa. Todos os direitos reservados.</p> <nav> <a href="#privacidade">Privacidade</a> </nav> </footer>

Elementos de Conteúdo

<figure> e <figcaption>

Propósito: Conteúdo ilustrativo com legenda

<figure> <img src="grafico.jpg" alt="Vendas 2024"> <figcaption>Gráfico de vendas 2024</figcaption> </figure>

<details> e <summary>

Propósito: Conteúdo expansível

Clique para expandir detalhes

Este conteúdo fica oculto até o utilizador clicar no summary. É útil para FAQs, informações adicionais, ou qualquer conteúdo que deve ser opcional.

  • Funciona sem JavaScript
  • Acessível por padrão
  • Pode ser estilizado com CSS
<details> <summary>FAQ: Como funciona?</summary> <p>Explicação detalhada...</p> </details>

<time>

Propósito: Datas e horas legíveis por máquinas

Exemplo: 15 de março de 2024, 14:30

<time datetime="2024-03-15">15 de março de 2024</time> <time datetime="2024-03-15T14:30:00">hoje às 14:30</time>

<mark>

Propósito: Destacar texto relevante

Este texto tem palavras destacadas para chamar atenção.

<p>Procurar por <mark>HTML semântico</mark> no texto.</p>

💡 Boas Práticas

Hierarquia de Cabeçalhos

Use h1-h6 de forma hierárquica:

  • h1 - Título principal (apenas um por página)
  • h2 - Seções principais
  • h3 - Subseções
  • h4-h6 - Níveis mais específicos

❌ Hierarquia incorreta

<h1>Título Principal</h1> <h3>Pula h2</h3> <h2>Volta a h2</h2> <h5>Pula para h5</h5>

✅ Hierarquia correta

<h1>Título Principal</h1> <h2>Seção Principal</h2> <h3>Subseção</h3> <h4>Sub-subseção</h4>

Acessibilidade

  • Use role e aria-label quando necessário
  • Forneça alternativas textuais para conteúdo visual
  • Mantenha ordem lógica de tabulação
  • Use contrastes adequados
  • Teste com leitores de ecrã

SEO e Performance

  • Use apenas um <h1> por página
  • Inclua palavras-chave em cabeçalhos relevantes
  • Use <main> para identificar conteúdo principal
  • Estruture conteúdo de forma lógica
  • Evite divs e spans desnecessários

🎯 Resumo

HTML semântico não é apenas sobre usar as tags certas - é sobre criar uma web mais acessível, compreensível e funcional para todos. Ao estruturar o conteúdo com significado, beneficiamos utilizadores, desenvolvedores e máquinas.
