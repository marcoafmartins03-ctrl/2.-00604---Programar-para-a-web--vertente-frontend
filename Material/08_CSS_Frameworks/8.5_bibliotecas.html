Lição 8.5: Bibliotecas de Componentes

Componentes prontos para acelerar o desenvolvimento com Tailwind

O que são bibliotecas de componentes?

São coleções de elementos de interface prontos a usar: botões, cards, formulários, navegação, modais, etc.

✓ Vantagens

  • • Desenvolvimento mais rápido
  • • Design consistente
  • • Testados e responsivos
  • • Acessibilidade incluída

⚠ Considerações

  • • Dependência externa
  • • Menos controlo visual
  • • Possível peso extra
  • • Curva de aprendizagem

Tipos de Bibliotecas

📋

1. Copy/Paste (só HTML)

Grátis

Copias o código HTML diretamente para o teu projeto. Não precisa de JavaScript - funciona só com classes Tailwind.

Exemplos:

  • • HyperUI - hyperui.dev (componentes variados)
  • • Meraki UI - merakiui.com (moderno, limpo)
  • • Mamba UI - mambaui.com (simples)
  • • Flowrift - flowrift.com (marketing)

Quando usar: Projetos simples, landing pages, sem interatividade complexa.

⚡

2. Com JavaScript

Requer JS

Componentes interativos: dropdowns, modais, tabs, accordions. Precisam de um script JS para funcionar.

Exemplos:

  • • Flowbite - flowbite.com (muito popular, grátis + pro)
  • • Preline - preline.co (dashboards, admin)
  • • Tailwind Elements - tw-elements.com (estilo Bootstrap)

Quando usar: Quando precisas de interatividade sem usar React/Vue.

⚛️

3. Para React/Vue

Grátis

Componentes para frameworks JavaScript modernos. Importas como componentes no teu projeto React/Vue.

Exemplos:

  • • Headless UI - headlessui.com (oficial Tailwind)
  • • shadcn/ui - ui.shadcn.com (React, muito popular)
  • • Radix UI - radix-ui.com (primitivas acessíveis)

Quando usar: Projetos React ou Vue com Tailwind.

🔌

4. Plugin Tailwind (DaisyUI)

Grátis

Instala-se como plugin no Tailwind. Adiciona classes semânticas como btn, card, navbar.

DaisyUI - daisyui.com

  • • Classes semânticas (como Bootstrap)
  • • 30+ temas incluídos
  • • Combina com classes Tailwind normais
  • • Muito popular e bem documentado

Quando usar: Se preferes classes tipo Bootstrap mas com Tailwind.

Comparação: O mesmo botão

Vê como se escreve um botão primário em cada abordagem:

Tailwind Puro (muitas classes)

<button class="bg-blue-500 hover:bg-blue-600
        text-white font-medium py-2 px-4
        rounded-lg transition">
  Clica aqui
</button>

DaisyUI (classe semântica)

<button class="btn btn-primary">
  Clica aqui
</button>

<!-- Requer plugin DaisyUI instalado -->

Flowbite (classes Tailwind organizadas)

<button type="button"
        class="text-white bg-blue-700 hover:bg-blue-800
        focus:ring-4 focus:ring-blue-300 font-medium
        rounded-lg text-sm px-5 py-2.5">
  Clica aqui
</button>

Exemplo Prático: Usar Flowbite

Passo 1: Incluir o script

<!-- No final do <body> -->
<script src="https://cdn.jsdelivr.net/npm/flowbite@2.4.1/dist/flowbite.min.js"></script>

Passo 2: Usar um componente (Dropdown)

<!-- Botão que abre o dropdown -->
<button data-dropdown-toggle="meuMenu"
        class="bg-blue-700 text-white px-4 py-2 rounded">
  Menu ▼
</button>

<!-- Conteúdo do dropdown (inicialmente oculto) -->
<div id="meuMenu" class="hidden z-10 bg-white rounded shadow w-44">
  <ul class="py-2 text-sm">
    <li><a href="#" class="block px-4 py-2 hover:bg-gray-100">Perfil</a></li>
    <li><a href="#" class="block px-4 py-2 hover:bg-gray-100">Definições</a></li>
    <li><a href="#" class="block px-4 py-2 hover:bg-gray-100">Sair</a></li>
  </ul>
</div>

Nota: O atributo data-dropdown-toggle liga o botão ao menu. O JavaScript do Flowbite trata do resto!

Guia Rápido: Qual escolher?

Situação Recomendação
Landing page simples, sem interação HyperUI ou Meraki UI (copy/paste)
Preciso de dropdown/modal/tabs Flowbite (grátis, bem documentado)
Dashboard ou painel admin Preline ou Windmill UI
Projeto React shadcn/ui ou Headless UI
Quero classes simples tipo Bootstrap DaisyUI (plugin)
Qualidade premium, suporte oficial Tailwind UI (pago)
